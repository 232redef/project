# MLP Library — Система управления книгами и авторами

## Описание проекта

Веб-приложение для управления библиотечным фондом, где можно создавать, редактировать, удалять и просматривать информацию об авторах и их книгах. Приложение разработано в рамках учебной практики для демонстрации навыков fullstack-разработки на современном стеке технологий.

Предметная область: Библиотека книг и авторы (у одного автора — много книг).

Связь сущностей: один-ко-многим (One-to-Many).

---

## Архитектура

Выбран вариант Б: Fullstack Next.js

- Frontend и Backend объединены в едином приложении Next.js (App Router);
- Серверная логика реализована через Route Handlers в директории `app/api/`;
- In-memory хранение данных (массивы в памяти сервера с возможностью сохранения в JSON-файл);
- Приложение запускается на одном порту (3000);
- Отсутствие проблем с CORS благодаря единому домену и порту.

### Обоснование выбора

Выбор варианта Б обусловлен следующими преимуществами:
- единая кодовая база упрощает разработку и поддержку;
- отсутствие проблем с CORS;
- быстрый запуск и развертывание;
- встроенная маршрутизация на основе файловой структуры;
- возможность использования серверных компонентов Next.js;
- упрощенный деплой на Vercel.

Данный подход оптимален для учебных проектов и небольших приложений, где не требуется разделение команд разработчиков.

| updatedAt | string (ISO) | Да | Нет | Дата обновления записи |

| isbn | string | Да | Да | ISBN книги |
| genre | string | Да | Нет | Жанр книги |

### Связи между сущностями

```
Author (1) -----< (M) Book
   id                    authorId (ссылается на Author.id)
```

## REST API

### Базовый URL
```
http://localhost:3000/api
```

### Формат ответов

Успешный ответ с пагинацией:
```json
{
  "items": [...],
  "total": 10,
  "page": 1,
  "pages": 2
}
```

Успешный ответ с одной записью:
```json
{
  "id": "a1",
  "name": "Лев Толстой",
  "email": "tolstoy@example.com",
  "books": [...]
}
```

Ошибка:
```json
{
  "error": "Автор с таким email уже существует"
}
```

### Эндпоинты для авторов (Authors)

| Метод | URL | Описание | Параметры |
|-------|-----|----------|-----------|
| GET | `/api/authors` | Получить список авторов (с пагинацией) | `?page=1&limit=5` |
| GET | `/api/authors/:id` | Получить автора по ID (с его книгами) | — |
| POST | `/api/authors` | Создать нового автора | Тело запроса (JSON) |
| PATCH | `/api/authors/:id` | Частично обновить автора | Тело запроса (JSON) |
| DELETE | `/api/authors/:id` | Удалить автора (если нет книг) | — |
| GET | `/api/authors/search` | Поиск авторов по текстовым полям | `?q=текст` |

### Эндпоинты для книг (Books)

| Метод | URL | Описание | Параметры |
|-------|-----|----------|-----------|
| GET | `/api/books` | Получить список книг | `?page=1&limit=5&q=текст&authorId=id` |
| GET | `/api/books/:id` | Получить книгу по ID (с автором) | — |
| POST | `/api/books` | Создать новую книгу | Тело запроса (JSON) |
| PATCH | `/api/books/:id` | Частично обновить книгу | Тело запроса (JSON) |
| DELETE | `/api/books/:id` | Удалить книгу | — |
| GET | `/api/books/search` | Поиск книг по текстовым полям | `?q=текст` |

## Пользовательский интерфейс

### Страницы приложения

| Маршрут | Описание |
|---------|----------|
| `/` | Главная страница с описанием и навигацией |
| `/authors` | Список авторов (таблица, пагинация) |
| `/authors/new` | Форма создания автора |
| `/authors/:id` | Детальная страница автора (с книгами) |
| `/authors/:id/edit` | Форма редактирования автора |
| `/books` | Список книг (карточки, поиск, фильтр) |
| `/books/new` | Форма создания книги |
| `/books/:id/edit` | Форма редактирования книги |

### Функциональные возможности

1. Авторы:
   - Просмотр списка авторов с пагинацией (5 записей на страницу)
   - Детальный просмотр автора со списком его книг
   - Создание нового автора с валидацией
   - Редактирование информации об авторе
   - Удаление автора (с проверкой наличия книг)

2. Книги:
   - Просмотр списка книг в виде карточек
   - Поиск книг по названию, жанру или ISBN (с debounce 300 мс)
   - Фильтрация книг по автору (выпадающий список)
   - Пагинация (5 книг на страницу)
   - Создание новой книги с выбором автора
   - Редактирование информации о книге
   - Удаление книги

3. Общее:
   - Адаптивный дизайн (поддержка экранов от 320px)
   - Индикация состояния загрузки
   - Отображение ошибок в понятном виде
   - Подтверждение действия перед удалением
   - Блокировка кнопки отправки формы во время запроса

## Запуск проекта

### Требования

- Node.js 18+
- npm или yarn

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone <url-репозитория>
cd mlp-library

# 2. Установить зависимости
npm install

# 3. Запустить в режиме разработки
npm run dev

# 4. Открыть в браузере
http://localhost:3000
```

### Доступные скрипты

| Команда | Описание |
|---------|----------|
| `npm run dev` | Запуск в режиме разработки (с hot reload) |
| `npm run build` | Сборка production-версии |
| `npm start` | Запуск production-версии |
| `npm run lint` | Проверка кода ESLint |
